# Agent Guidance — Another-Intelligence (Cerebro)

## What This Repo Does
Another-Intelligence is the cognitive core. It runs the PPAC decision loop (Proposer → Predictor → Accumulator → Actor → Critic) with real RPE-driven learning. It hosts three permanent Ollama model roles and discovers all external tooling via MCP clients.

## Current State
- Full PPAC loop is implemented and tested (brain.py, strategist/, executor/, reflex/, rpe/).
- MCP client discovers and calls ADHD, ASD, and OCD servers at runtime.
- Persistent memory (episodic, semantic, value-index) is functional.
- Statusline visualizes brain-region activity.
- CLI commands: `ai brain`, `ai status`, `ai mcp list`.

## MCP Servers Available in This Workspace
This `.mcp.json` registers three MCP servers:
- **adhd** — Attention Deficit Hyperactivity Driver (attention-deficit-hyperactivity-driver/). Use for signin, heartbeat, posting schemas/dependencies/status, and reading bus activity.
- **asd** — Autism Spectrum Driver (autism-spectrum-driver/). Use for knowledge ingestion, compilation, semantic query, and validation.
- **ocd** — Obsessive Compulsive Driver (obsessive-compulsive-driver/). Use before every push for quality gates, lint, format check, and secret scanning.

When starting work, call `adhd_signin`. When blocked or ready for review, call `adhd_post(type="status", topic="agent-activity")`. When finished, call `adhd_signout`.

## MCP Change Coordination (Critical)
When you modify MCP server code in ANY of the three drivers (adhd, asd, or ocd), you MUST notify other active sessions via the ADHD bus BEFORE and AFTER the change:

1. **Before change**: Post `adhd_post(type="event", topic="mcp-change", payload={"server": "<name>", "action": "preparing", "branch": "<branch>", "session_id": "<id>"})`
2. **Wait**: Allow ~5 seconds for other sessions to see the notification and pause tool calls to that server.
3. **After change (server ready)**: Post `adhd_post(type="event", topic="mcp-change", payload={"server": "<name>", "action": "ready", "commit": "<hash>", "session_id": "<id>"})`
4. **If you see another session's "preparing" notification**: Pause all MCP tool calls to that server until you see the matching "ready" message.
